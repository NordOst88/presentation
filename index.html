A brief history of programming languages

Genealogy of Common Languages

TIOBE

Zuse’s Plankalkül (1945)

  1. Never implemented
  2. Advanced data structures
    • floating point, arrays, records
  3. Invariants

Pseudocodes

  1. Short code; 1949; BINAC; Mauchly
    • Expressions were coded, left to right

  2. Speedcoding; 1954; IBM 701, Backus
    • Pseudo ops for arithmetic and math functions
    • Conditional and unconditional branching
    • Autoincrement registers for array access
    • Slow!
    • Only 700 words left for user program

FORTRAN (mid-late 1950’s)

  1. Emphasis on scientific programming
  2. FORTRAN 0 (not implemented) - 1954
  3. FORTRAN I - 1957
  4. FORTRAN II - 1958
  5. FORTRAN IV - 1960-62
  6. FORTRAN 77 - 1978
  7. FORTRAN 90 - 1990
  8. FORTRAN 95 - 1995

Dramatically changed forever the way computers are used

FORTRAN in depth

  1. FORTRAN II – 1958
    • Independent compilation
  2. FORTRAN IV – 1960-62
    • Improved data typing
    • ANSI standard in 1966
  3. FORTRAN 77 – 1978
    • Better logical statements
  4. FORTRAN 90 – 1990
    • Allows recursion
    • Syntactic changes
  5. FORTRAN 95 – 1995
    • Deletes some deprecated features
    • Deprecates more features

LISP (1959)

  1. Designed at MIT by McCarthy
  2. Still the dominant language for AI
  3. Pioneered functional programming
  4. Syntax is based on lambda calculus
  5. Contemporary dialects: COMMON LISP, Scheme
  6. Object-oriented versions: Flavors, LOOPS, CLOS
  7. Related languages: ML, Miranda, Haskell

Scheme

  1. Developed at MIT in mid 1970s
  2. Small
  3. Extensive use of static scoping
  4. Functions as first-class entities
  5. Simple syntax (and small size) make it ideal for educational applications

COMMON LISP

  1. An effort to combine features of several dialects of LISP into a single language
  2. Large, complex

ALGOL

  1. Goals of the language:
    • Close to mathematical notation
    • Good for describing algorithms
    • Must be translatable to machine code

  2. Not meant to be implemented, but variations of it were - MAD, JOVIAL
  3. IBM initially enthusiastic, but all support dropped by mid-1959

ALGOL 58 (1958)

  1. Formalized the concept of the data type
  2. Compound statements (begin ... end)
  3. Parameters were separated by mode (in & out)
  4. Names could have any length
  5. Arrays could have any number of subscripts
  6. Subscripts were placed in brackets
  7. Semicolon as a statement separator
  8. Assignment operator was :=
  9. if had an else-if clause
  10. no I/O - “would make it machine dependent”

ALGOL 60 (1960)

  1. New Features:
    • Block structure; multiple scopes
    • Better parameter passing
    • Recursion
    • Stack-dynamic arrays
  2. Successes:
    • Standard for publishing algorithms for over 20 years
    • All subsequent imperative languages are based on it
    • First machine-independent language
    • First formally defined syntax
  3. Failures:
    • Programs non-portable
    • Hard to implement
    • Intrenchment of FORTRAN
    • Formal syntax description
    • Lack of support of IBM

COBOL (1960)

  1. Based on FLOW-MATIC, which had:
    • Names up to 12 characters, with embedded hyphens
    • English names for arithmetic operators
    • Data and code kept completely separate
    • Verbs as the first word in every statement
  2. COBOL Design Problems:
    • Arithmetic expressions
    • Subscripts
    • Fights among manufacturers
  3. Contributions:
    • Macros
    • Hierarchical data structures
    • Nested selection statements
    • Long names with hyphens
    • Separate data division

BASIC (1964)

  1. Intended for teaching how to program
  2. Interactive interpreted environment
  3. Designed by Kemeny & Kurtz at Dartmouth
  4. First widely used language with time sharing
  5. Current popular dialect: Visual BASIC

PL/I (1965)

  1. Contributions:
    • First unit-level concurrency
    • First exception handling
    • Switch-selectable recursion
    • First pointer data type
    • First array cross sections
  2. Comments:
    • Many new features were poorly designed
    • Too large and too complex
    • Was (and still is) actually used for both scientific and business applications

APL (1962)

  1. Array and matrix processing
  2. Dynamic typing and dynamic storage allocation
  3. Designed as a hardware description language
  4. Highly expressive
  5. Programs are very difficult to read

SIMULA 67 (1967)

  1. Introduced classes - data and operations together
  2. Designed primarily for system simulation
  3. Based on ALGOL 60 and SIMULA I
  4. Primary Contribution: Classes, objects, and inheritance

ALGOL 68 (1968)

  1. Primary design goal: orthogonality
  2. Based on ALGOL 60, but not a superset

Pascal (1971)

  1. Designed for teaching structured programming
  2. Small, simple, nothing really new
  3. Many subsequent languages based on Pascal:
    • Delphi
    • Occam

C (1972)

  1. Designed for systems programming
  2. Powerful set of operators
  3. Poor type checking
  4. Initially spread through UNIX
  5. Flexible, but unsafe

Prolog (1972)

  1. Logic language, non-procedural
  2. Programs are sets of facts and rules
  3. Not proven to be widely applicable

Ada 1983 (began in mid-1970s)

  1. Contributions:
    • Packages; Exception handling; Generic program units; Concurrency
  2. Comments:
    • Competitive design
    • Included all that was then known about software engineering and language design
    • First compilers were very difficult: Five years to develop the first really usable compiler
  3. Ada 95 (began in 1988)
    • Support for OOP through type derivation
    • New concurrency features
    • More flexible libraries

Smalltalk (1972-1980)

  1. Object-oriented programming language
  2. Really a graphical environment
  3. Developed at Xerox PARC
  4. First full implementation of an object-oriented language
  5. Pioneered the GUI everyone now uses

C++ (1985)

  1. Design goal: no performance penalty, relative to C
  2. Features:
    • Parameterized types (templates)
    • Exception handling
    • Operator overloading
  3. Facilities for object-oriented programming added to C
  4. Large and complex
  5. Rapidly grew in popularity, along with OOP
  6. ANSI standard approved in November, 1997

Java (early 1990’s)

  1. Developed at Sun Microsystems
    • Smaller, simpler, more reliable C++
    • Significantly simplified
    • All code must be encapsulated in classes
    • Concurrency support (threads)
    • Garbage collected
  2. JavaScript (1995)
    • Began as LiveScript at Netscape
    • Client-side HTML-resident scripting language
    • Used for designing client-side dynamic documents
    • Purely interpreted

Scripting Languages

  1. Earliest scripts were files which stored a sequence of OS commands
    • interpreted by simple OS facility
  2. UNIX had a profound influence
    • sh
    • ksh
    • awk
  3. Not all are Turing Complete

Perl (1987)

  1. Related to ALGOL only through C
  2. A scripting language
    • A script (file) contains instructions to be executed
    • Other examples: sh, awk, tcl/tk
  3. Developed by Larry Wall
  4. Perl variables are statically typed and implicitly declared
    • Three distinctive namespaces, denoted by the first character of a variable’s name
  5. Powerful but somewhat dangerous
  6. Widely used as a general purpose language

Scripting Languages for the Web

  1. JavaScript
    • Used in Web programming (client side) to create dynamic HTML documents
  2. PHP (1994)
    • Server-side HTML-resident scripting language
    • Purely interpreted
  3. Python
    • An OO interpreted scripting language

C# (2000)

  1. Microsoft
  2. The primary language of the .NET platform
  3. A successor to both Java and C++
  4. Includes most of the features of Java, with some modifications, as well as some of the features of C++ that were left out of Java
  5. Can be used for both .NET applications for the Web, as well as a general-purpose programming language

Markup/Programming Hybrid Languages

  1. XML: eXtensible Markup Language
    • a metamarkup language
    • eXtensible Stylesheet Language Transformation (XSTL) transforms XML documents for display
  2. JSP: Java Server Pages
    • a collection of technologies to support dynamic Web documents
